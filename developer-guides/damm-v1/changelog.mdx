<Update
  label="@meteora-ag/dynamic-amm-sdk 1.3.9"
  description="Newer lock fee claim helper"
  rss="DAMM v1 SDK 1.3.9 adds claimLockFee2 and simplifies claimLockFee behavior."
>
  ## Release Summary

  | Component | Version | PR |
  | --- | --- | --- |
  | TypeScript SDK | `1.3.9` | [dynamic-amm-sdk #222](https://github.com/MeteoraAg/dynamic-amm-sdk/pull/222) |

  ## Changed

  - Removed ternary logic in `claimLockFee`.
  - Added `claimLockFee2` to claim lock fees without `tempWSolAcc`.

  ## Integration Impact

  | Integrator | What To Check |
  | --- | --- |

</Update>

<Update
  label="dynamic-amm 0.6.1"
  description="Customizable pools and partner fees"
  rss="DAMM v1 dynamic-amm 0.6.1 adds customizable constant-product pool creation and partner fee claiming, with PartnerInfo added to Pool state."
>
  ## Release Summary

  | Component | Version | PR |
  | --- | --- | --- |
  | Program crate | `0.6.1` | [mercurial dynamic AMM #180](https://github.com/mercurial-finance/mercurial-dynamic-amm-sdk/pull/180) |
  | TypeScript SDK | `1.1.19` | [mercurial dynamic AMM #180](https://github.com/mercurial-finance/mercurial-dynamic-amm-sdk/pull/180) |

  ## Added

  - Added `initialize_customizable_permissionless_constant_product_pool`.
  - Added `partner_claim_fees`.
  - Added SDK `createCustomizablePermissionlessConstantProductPool`.
  - Added SDK `partnerClaimFees`.

  ## Changed

  - `create_config` now requires `partner_fee_numerator`.
  - `Pool` state includes `PartnerInfo` for partner fee sharing.

  ## Integration Impact

  | Integrator | What To Check |
  | --- | --- |
  | Pool creators | Pass partner fee parameters when creating configs. |
  | Indexers | Decode `PartnerInfo` from pool state. |
  | Fee claimers | Separate protocol fee balances from partner pending fee amounts. |
</Update>
