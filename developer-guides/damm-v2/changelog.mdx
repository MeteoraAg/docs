  <Warning>
    **Breaking integration area.** Re-test custom quote engines, event decoders, and pool creation code if you support compounding pools or parse `EvtSwap2` fee fields.
  </Warning>

  ## Release Summary

  | Component | Version | Notes |
  | --- | --- | --- |
  | On-chain program | `cp_amm 0.2.0` | Adds compounding collect fee mode and tracked reserves |
  | TypeScript SDK | `1.4.3` | Fixes `createPool`, `createCustomPool`, and `createCustomPoolWithDynamicConfig` liquidity handling by collect fee mode |

  ## What Changed

  | Area | Change | Developer action |
  | --- | --- | --- |
  | Collect fee mode | Added `CollectFeeMode.Compounding` (`2`) | Pass `collectFeeMode` into quote and pool creation helpers |
  | Pool layout | New pools track `token_a_amount` and `token_b_amount` in layout `V1` | Use current SDK quote helpers for reserve-aware calculations |
  | Swap event | `partner_fee` replaced by `compounding_fee` in `EvtSwap2.swap_result` | Update event decoders and analytics schemas |
  | Fee result types | SDK split fees into `claimingFee` and `compoundingFee` | Replace old `tradingFee`/`partnerFee` assumptions |
  | Pool creation | Compounding pools subtract `DEAD_LIQUIDITY` differently from concentrated pools | Upgrade to SDK `1.4.3+` |

  ## Upgrade Checklist

  1. Upgrade TypeScript integrations to `@meteora-ag/cp-amm-sdk@1.4.3` or later.
  2. Use `getQuote2`/`swap2` for new swap integrations.
  3. Update event decoders to read `EvtSwap2.swap_result.compounding_fee`.
  4. Include `collectFeeMode`, current reserves, and liquidity in deposit/withdraw quote paths where the SDK type requires them.
  5. Re-run quote tests for concentrated and compounding pools.
  6. For custom generated clients, refresh from the current public IDL.
</Update>
