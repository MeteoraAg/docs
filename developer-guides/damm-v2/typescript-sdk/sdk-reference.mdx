This page summarizes the SDK surface. For exhaustive TypeScript types, use the public source and generated declarations in the [`damm-v2-sdk`](https://github.com/MeteoraAg/damm-v2-sdk) repository.

## Core Exports

| Export | Use |
| --- | --- |
| `CpAmm` | Main class for fetching state, quoting, and building transactions |
| `CpAmmIdl` / `CpAmmTypes` | Published IDL JSON and generated Anchor type |
| `CP_AMM_PROGRAM_ID` | Program ID constant |
| `CollectFeeMode` | `BothToken`, `OnlyB`, `Compounding` |
| `BaseFeeMode` | Time scheduler, rate limiter, and market-cap scheduler modes |
| `ActivationType` | `Slot` or `Timestamp` activation points |
| `SwapMode` | `ExactIn`, `PartialFill`, `ExactOut` |
| `MIN_SQRT_PRICE`, `MAX_SQRT_PRICE` | Global price bounds |
| `DEAD_LIQUIDITY` | Compounding-pool dead liquidity constant, `100 << 64` |

## State Methods

| Method | Notes |
| --- | --- |
| `fetchConfigState(config)` | Fetch one static or dynamic config account |
| `fetchPoolState(pool)` | Fetch one pool account |
| `fetchPoolFees(pool)` | Decode pod-aligned pool fee data |
| `fetchPositionState(position)` | Fetch one position account |
| `getStaticConfigs()` | Fetch public static configs |
| `getAllPools()` | RPC scan for pool accounts; prefer API for broad discovery in apps |
| `getAllPositionsByPool(pool)` | RPC scan for pool positions |
| `getPositionsByUser(user)` | Resolve position NFTs owned by a wallet and fetch positions |
| `getAllVestingsByPosition(position)` | Fetch external vesting accounts for a position |

## Quote Methods

| Method | Use |
| --- | --- |
| `getQuote` | Legacy exact-in quote path |
| `getQuote2` | Exact-in, partial-fill, and exact-out quotes; prefer for new swap integrations |
| `getDepositQuote` | Convert a token input amount into liquidity and counterpart amount |
| `getWithdrawQuote` | Convert liquidity into expected token A and token B outputs |
| `preparePoolCreationParams` | Compute initial square-root price and liquidity for pool creation |
| `preparePoolCreationSingleSide` | Prepare one-sided concentrated pool initialization |

When using compounding pools, pass `collectFeeMode: CollectFeeMode.Compounding` and include current `tokenAAmount`, `tokenBAmount`, and `liquidity` where quote helpers require them.

## Fee And Price Helpers

Use SDK helpers instead of hand-encoding fee bytes:

| Helper | Use |
| --- | --- |
| `getBaseFeeParams` | Encode base fee parameters for pool creation |
| `getDynamicFeeParams` | Encode default dynamic fee parameters from max dynamic fee bps |
| `bpsToFeeNumerator`, `feeNumeratorToBps` | Convert between bps and DAMM v2 fee numerators |
| `decodeFeeTimeSchedulerParams`, `decodeFeeRateLimiterParams`, `decodeFeeMarketCapSchedulerParams` | Decode Borsh event fee bytes |
| `decodePodAlignedFeeTimeScheduler`, `decodePodAlignedFeeRateLimiter`, `decodePodAlignedFeeMarketCapScheduler` | Decode pool-account fee bytes |
| `getSqrtPriceFromPrice`, `getPriceFromSqrtPrice` | Convert UI prices and Q64 square-root prices |

<Warning>
  The same base-fee mode uses different byte layouts in initialize events and pool accounts. Decode `EvtInitializePool` base fee data with Borsh decoders and pool-state base fee data with pod-aligned decoders.
</Warning>

## Gotchas

| Topic | What to remember |
| --- | --- |
| Token order | Pool PDAs sort mints, but transaction params still need the pool's `tokenAMint` and `tokenBMint` fields. |
| Compounding mode | `CollectFeeMode.Compounding` uses full-range constant-product liquidity and splits swap fees into `claimingFee` and `compoundingFee`. |
| Rate limiter | Low-level swap builders must include the instructions sysvar remaining account. The SDK handles this when it can identify the pool mode. |
| Layout version | New pools are layout version `V1` and track `tokenAAmount` and `tokenBAmount`. Quote code should use current SDK versions for compounding pools. |
| Events | Prefer `EvtSwap2` for swap indexing. Older swap events were deprecated and removed from newer program versions. |
