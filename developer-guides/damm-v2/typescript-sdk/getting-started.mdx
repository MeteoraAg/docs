Use the TypeScript SDK for most DAMM v2 integrations. It wraps the public IDL, derives PDAs, fetches account state, calculates quotes, handles Token 2022 transfer-fee adjustments, and builds transactions for pools, swaps, liquidity, positions, locks, and rewards.

## Install

```bash
npm install @meteora-ag/cp-amm-sdk @solana/web3.js @coral-xyz/anchor bn.js
```

```bash
pnpm add @meteora-ag/cp-amm-sdk @solana/web3.js @coral-xyz/anchor bn.js
```

```bash
yarn add @meteora-ag/cp-amm-sdk @solana/web3.js @coral-xyz/anchor bn.js
```

The local SDK reference used for this guide is `@meteora-ag/cp-amm-sdk@1.4.3`.

## Initialize

```typescript
import { Connection, PublicKey } from "@solana/web3.js";
import { CpAmm } from "@meteora-ag/cp-amm-sdk";

const connection = new Connection("https://api.mainnet-beta.solana.com", "confirmed");
const cpAmm = new CpAmm(connection);

const pool = new PublicKey("POOL_ADDRESS");
const poolState = await cpAmm.fetchPoolState(pool);
```

## Common Flow

| Task | Prefer |
| --- | --- |
| Fetch config or pool state | `fetchConfigState`, `fetchPoolState`, `fetchPoolFees` |
| Discover pools or positions | `getAllPools`, `getAllPositionsByPool`, `getPositionsByUser`, or the DAMM v2 API |
| Quote a swap | `getQuote2` for exact-in, partial-fill, and exact-out support |
| Build a swap | `swap2` for new integrations; `swap` is exact-in only |
| Create pools | `createPool`, `createCustomPool`, or `createCustomPoolWithDynamicConfig` |
| Add/remove liquidity | `getDepositQuote`, `addLiquidity`, `getWithdrawQuote`, `removeLiquidity` |
| Manage position ownership | `createPosition`, `lockPosition`, `permanentLockPosition`, `splitPosition2`, `closePosition` |
| Rewards | `initializeReward`, `fundReward`, `claimReward`, `withdrawIneligibleReward` |

## Quote Before You Build

```typescript
import { BN } from "@coral-xyz/anchor";
import { SwapMode } from "@meteora-ag/cp-amm-sdk";

const quote = cpAmm.getQuote2({
  inputTokenMint,
  poolState,
  currentPoint:
    poolState.activationType === 0
      ? new BN(await connection.getSlot())
      : new BN(Math.floor(Date.now() / 1000)),
  slippage: 0.5,
  tokenADecimal: 9,
  tokenBDecimal: 6,
  hasReferral: false,
  swapMode: SwapMode.ExactIn,
  amountIn: new BN(1_000_000),
});
```

Use the returned `minimumAmountOut` or `maximumAmountIn` in the transaction builder. For Token 2022 transfer-fee tokens, pass the mint information fields supported by the quote helpers when you need transfer-fee-aware output.

## Build Transactions

SDK transaction methods return a `Transaction` or transaction builder promise. You still choose the payer, recent blockhash, signing flow, and submission path for your app.

```typescript
const tx = await cpAmm.swap2({
  payer: wallet.publicKey,
  pool,
  inputTokenMint,
  outputTokenMint,
  tokenAMint: poolState.tokenAMint,
  tokenBMint: poolState.tokenBMint,
  tokenAVault: poolState.tokenAVault,
  tokenBVault: poolState.tokenBVault,
  tokenAProgram,
  tokenBProgram,
  referralTokenAccount: null,
  poolState,
  swapMode: SwapMode.ExactIn,
  amountIn: new BN(1_000_000),
  minimumAmountOut: quote.minimumAmountOut!,
});
```

<Note>
  Rate-limiter pools require the instructions sysvar account on low-level swap builders. The TypeScript SDK detects rate-limiter pools in `swap` and `swap2` when it has `poolState`; pass `poolState` to avoid an extra RPC read and to keep account planning explicit.
</Note>

## Public Testing

The SDK repository includes public Vitest and Bankrun tests. They use the public SDK source and a local program fixture, so they are best for SDK contributors and regression testing. App integrators should also test against devnet or their own local validator flow before shipping transaction code.

```bash
git clone https://github.com/MeteoraAg/damm-v2-sdk.git
cd damm-v2-sdk
pnpm install
pnpm test
```
