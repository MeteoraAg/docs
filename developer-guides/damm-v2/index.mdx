## Recommended Build Path

1. Read the [product overview](/core-products/damm-v2/what-is-damm-v2) only far enough to understand pool modes, fees, positions, and Token 2022 support.
2. Use the [TypeScript SDK](/developer-guides/damm-v2/typescript-sdk/getting-started) for transactions and quotes unless you specifically need CPI or a generated client.
3. Review [accounts](/developer-guides/damm-v2/program/accounts), [instructions](/developer-guides/damm-v2/program/instructions), and [errors](/developer-guides/damm-v2/program/errors) before building custom indexers, generated clients, or low-level integrations.
4. Use [Rust CPI](/developer-guides/damm-v2/cpi/integration) for on-chain composition and [Rust SDK](/developer-guides/damm-v2/rust-sdk/integration) for off-chain Rust quote math.
5. Use the [API reference](/developer-guides/damm-v2/api-reference/overview) for indexed data, charts, and discovery. Use on-chain state or SDK helpers for transaction-critical reads.
6. Check the [developer changelog](/developer-guides/damm-v2/changelog) before upgrading program, SDK, or event decoders.
