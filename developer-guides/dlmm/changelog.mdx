  ## Who needs to act

  | Integration | Why | What to do |
  | --- | --- | --- |
  | **Apps using the TypeScript SDK** | New APIs, quote changes, removed types | Upgrade to `@meteora-ag/dlmm@1.9.5+`, re-test swaps and pool creation |
  | **Trading terminals (swap UI)** | Swap quote math and max bins per instruction changed | Re-run quote tests; split large routes if you hit the 260-bin cap |
  | **Indexers / analytics** | `swap` / `swap2` behavior and events changed | Index **`Swap2Evt`** (not legacy swap events); decode `FeeParameterUpdate.base_fee_power_factor` |
  | **Low-level instruction builders** | Account mutability and remaining-accounts layout changed | Mark bitmap extension **writable** on `swap` / `swap2`; add `TransferHookReferral` when using `host_fee_in` |
  | **Pool / preset creators** | New fields on initialize and preset accounts | Pass `concreteFunctionType` and `collectFeeMode` when creating pairs or presets |
  | **LM-only integrations** | Legacy pools without rewards become limit-order pools | Read `functionType` per pair before enabling rewards or limit orders |

  ## Upgrade checklist

  Pick the path that matches your stack. Complete every item that applies.

  ### All integrations

  1. Pin versions: program `0.12.0`, SDK `1.9.5`, CLI `0.6.3`, commons `0.3.3` (or later).
  2. Regenerate or refresh clients from the [published IDL](https://github.com/MeteoraAg/dlmm-sdk/blob/main/idls/dlmm.json) if you do not use the official SDK.
  3. For each pool you touch, resolve **function type** (`Undetermined` → LM if rewards exist, else limit order) before calling LM or limit-order APIs.

  ### TypeScript SDK (`@meteora-ag/dlmm`)

  ```bash
  npm install @meteora-ag/dlmm@1.9.5 @solana/web3.js
  ```

  1. Replace removed `Position` (v1) usage — v1 positions no longer exist on chain.
  2. Remove calls to `updateBinArray` (removed).
  3. Re-test `swapQuote`, `swapQuoteExactOut`, and any code using `swapExactOutQuoteAtBin` (`amountIn` now includes fees).
  4. When creating pools, pass `concreteFunctionType` and `collectFeeMode` to `createCustomizablePermissionlessLbPair`.
  5. When looking up pools, pass the same filters to `getPairPubkeyIfExists` if you need a specific pair variant.
  6. For limit orders: use `placeLimitOrder`, `cancelLimitOrder`, `closeLimitOrderIfEmpty`, `getLimitOrder`, `getLimitOrderByUserAndLbPair`, `quoteCreateLimitOrder` — only on **limit-order** pairs.

  ### Trading terminals and custom quote engines

  1. Re-validate quotes for: `swap`, `swap_exact_out`, `swap_with_price_impact`, `swap2`, `swap_exact_out2`, `swap_with_price_impact2`.
  2. Account for **limit order liquidity** and **collect fee mode** (`InputOnly` vs `OnlyY`) in quote math — use SDK `1.9.5+` or commons `0.3.3+` rather than reimplementing.
  3. Cap or split routes: max bins per swap instruction is **260** (was 280).
  4. If you use `host_fee_in` on `swap2` / `swap_exact_out2` / `swap_with_price_impact2`, update `RemainingAccountsInfo` (see [Breaking changes](#breaking-changes-program)).

  ### Indexers and event pipelines

  1. Subscribe to **`Swap2Evt`** for swap activity on DLMM pools (pivot away from legacy swap event indexing where applicable).
  2. Extend `FeeParameterUpdate` decoders with `base_fee_power_factor`.
  3. Treat bitmap extension accounts on `swap` / `swap2` as **state-changing** (writable) in your account metadata.

  ### Low-level / CPI builders

  1. Pass bitmap extension as **mutable** on `swap` and `swap2`.
  2. Include `AccountsType::TransferHookReferral` in `RemainingAccountsInfo` when the instruction has `host_fee_in`, plus any transfer-hook accounts required for the referral mint.
  3. Update `initialize_preset_parameter` builders: `collect_fee_mode` + `concrete_function_type` (replaces the old `function_type` input).

  ## Program: `lb_clmm` `0.12.0`

  Program ID (mainnet and devnet): `LBUZKhRxPF3XUpBCjp4YzTKgLccjZhTSDM9YuVaPwxo`

  ### Added

  | Instruction / account | When to use |
  | --- | --- |
  | `add_liquidity_by_weight2` | Same as `add_liquidity_by_weight` but supports **Token-2022** mints. Prefer for new Token-2022 integrations. |
  | `set_permissionless_operation_bits` | Position owner enables permissionless ops (e.g. permissionless `claim_fee`). |
  | `place_limit_order` | Place bid or ask limit orders on limit-order pools. |
  | `cancel_limit_order` | Cancel an active limit order. |
  | `close_limit_order_if_empty` | Close the limit order account when all bins are empty. |
  | `close_bin_array` | Close an invalid or empty bin array when eligible. |
  | `PresetParameter.collect_fee_mode` | New field; existing accounts default to **`CollectFeeMode::InputOnly`**. |
  | `LbPair::StaticParameters.collect_fee_mode` | New field; existing pairs default to **`CollectFeeMode::InputOnly`**. |

  **Collect fee mode (on chain):**

  | Mode | Meaning |
  | --- | --- |
  | `InputOnly` (0) | Fees collected from the swap **input** token (previous default behavior). |
  | `OnlyY` (1) | Fees collected in **token Y** only (quote-token / single-sided fee collection). |

  ### Changed

  - **Permissionless `claim_fee`:** allowed when the position owner set the corresponding permissionless operation bit.
  - **Existing pairs without LM rewards:** migrated to **limit-order** pools; they **no longer support LM**. This is **not reversible**.
  - **`initialize_permissioned_lb_pair`:** `InitPermissionPairIx.protocol_share` is unused (treated as padding).
  - **`go_to_a_bin`:** reverts if the pool is disabled or before activation. Operators can bypass by passing operator-associated accounts in remaining accounts.

  ### Removed

  - `reset_pool_tombstone_fields`
  - `reset_bin_array_tombstone_fields`
  - `reset_position_tombstone_fields`

  <a id="breaking-changes-program" />

  ### Breaking changes (program)

  | Area | What changed | What you should do |
  | --- | --- | --- |
  | **Pool function type** | `FunctionType` no longer defaults to `LiquidityMining`. Existing `LbPair` accounts are `Undetermined`. At runtime: if LM rewards are initialized → behaves as **LM**; otherwise → **limit order**. | Before LM rewards or limit-order flows, read `functionType` (or use SDK helpers). Do not assume all legacy pools are LM. |
  | **Swap quotes** | Collect fee mode + limit order liquidity change quote results. | Affects `swap`, `swap_exact_out`, `swap_with_price_impact`, `swap2`, `swap_exact_out2`, `swap_with_price_impact2`. Upgrade SDK/commons or re-derive quotes. |
  | **Max bins per swap** | Limit order logic uses more CU. | Max bins per instruction: **280 → 260**. Split swaps or reduce per-tx bin traversal. |
  | **`swap` / `swap2` accounts** | Bitmap extension must be **mutable**. | Limit orders flip liquidity tracking bits during swaps. Mark the bitmap extension account writable in custom builders. |
  | **`FeeParameterUpdate` event** | New field `base_fee_power_factor`. | Update event decoders and downstream fee logic. |
  | **`initialize_preset_parameter`** | Requires `collect_fee_mode`; uses `concrete_function_type` instead of `function_type`. | Update preset-creation transactions and IDL clients. |
  | **`RemainingAccountsInfo`** | New `AccountsType::TransferHookReferral`. | For `swap2`, `swap_exact_out2`, `swap_with_price_impact2` **with `host_fee_in`**: append `TransferHookReferral` to the slice and include transfer-hook accounts for the **referral mint**. |

  **Function type reference (SDK):**

  ```typescript
  enum FunctionType {
    Undetermined = 0,      // Resolved at runtime: LM if rewards exist, else limit order
    LiquidityMining = 1,
    LimitOrder = 2,
  }

  enum ConcreteFunctionType {
    LimitOrder = 0,        // Use when creating new limit-order pools
    LiquidityMining = 1,
  }
  ```

  ## TypeScript SDK: `@meteora-ag/dlmm` `1.9.5`

  [SDK PR #273](https://github.com/MeteoraAg/dlmm-sdk/pull/273)

  ### Added

  | API | Use for |
  | --- | --- |
  | `placeLimitOrder` | Build a transaction to place bid/ask limit orders (limit-order pairs only). |
  | `cancelLimitOrder` | Cancel selected bins in a limit order. |
  | `closeLimitOrderIfEmpty` | Close an empty limit order account. |
  | `enablePositionPermissionlessClaimFee` | Let the position owner allow others to call `claim_fee`. |
  | `getLimitOrderByUserAndLbPair` | List a wallet's limit orders for a pair. |
  | `getLimitOrder` | Fetch one limit order by pubkey. |
  | `addLiquidityByWeight2` | Token-2022 weight-based liquidity. |
  | `quoteCreateLimitOrder` | Estimate cost to open a limit order. |

  ### Breaking changes

  | API | Change | Action |
  | --- | --- | --- |
  | `createCustomizablePermissionlessLbPair` | New args: `concreteFunctionType`, `collectFeeMode`. | Pass `ConcreteFunctionType.LimitOrder` or `.LiquidityMining` and `CollectFeeMode.InputOnly` or `.OnlyY`. |
  | `getPairPubkeyIfExists` | New optional filters: `concreteFunctionType`, `collectFeeMode`. | Pass filters when multiple pair variants can share the same mint pair. |

  ### Changed

  - `swapQuote` and `swapQuoteExactOut` include limit order liquidity and collect fee mode.
  - `swapExactOutQuoteAtBin`: returned `amountIn` **includes fees** (was excluding fees in some cases).

  ### Removed

  - `Position` type (v1) — no v1 positions remain on chain.
  - `updateBinArray` — unused; remove call sites.

  <Tip>
    RC for early testing: [`@meteora-ag/dlmm@1.9.5-rc.3`](https://www.npmjs.com/package/@meteora-ag/dlmm/v/1.9.5-rc.3). Prefer stable **`1.9.5`** for production.
  </Tip>

  **Example: create a limit-order pool with input-only fees**

  ```typescript
  import DLMM, {
    CollectFeeMode,
    ConcreteFunctionType,
  } from "@meteora-ag/dlmm";

  // When calling createCustomizablePermissionlessLbPair, pass for example:
  const concreteFunctionType = ConcreteFunctionType.LimitOrder;
  const collectFeeMode = CollectFeeMode.InputOnly;
  ```
