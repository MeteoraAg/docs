This page summarizes the account model used by `lb_clmm`. For exact layouts, use the IDL and generated TypeScript types exported by `@meteora-ag/dlmm`.

## Account Summary

| Account | Purpose |
| --- | --- |
| `LbPair` | Pool state: mints, reserves, active bin, fee parameters, reward info, oracle, bitmap, status, activation settings, creator, pair type, and Token-2022 flags. | 
| `BinArray` | Container for consecutive bins. Each bin stores token amounts, liquidity supply, fee state, reward state, and limit order counters. |
| `BinArrayBitmapExtension` | Extends the pool bitmap for initialized bin arrays outside the default range. | 
| `PositionV2` | LP position state for an owner and bin range. Stores shares, fee/reward checkpoints, operator, fee owner, lock release point, and permission bits. | 
| `LimitOrder` | Dynamic owner account for limit order bins. | 
| `Oracle` | Observation account used for pool oracle reads. |
| `PresetParameter` | Legacy fee/preset configuration for pool creation. | 
| `PresetParameter2` | Current pool creation preset with `bin_step`, `base_factor`, `base_fee_power_factor`, function type, and collect fee mode. | 
| `TokenBadge` | Admin allowlist account for Token-2022 mints that require explicit approval. | 
| `ClaimFeeOperator` / `Operator` | Operator/admin accounts for restricted flows. | 

## LbPair

`LbPair` is the root pool account. It includes:

- `parameters` and `v_parameters` for static and variable fee calculations.
- `active_id` and `bin_step` for the current price bin and price spacing.
- `token_x_mint`, `token_y_mint`, `reserve_x`, and `reserve_y`.
- `reward_infos`, with up to two reward vaults and funders.
- `oracle`, `bin_array_bitmap`, and optional `BinArrayBitmapExtension`.
- `pair_type`, `status`, `activation_type`, `activation_point`, and `pre_activation_duration`.
- `creator`, `creator_pool_on_off_control`, and Token-2022 program flags.

## BinArray And Bins

A `BinArray` covers a fixed sequence of bin IDs. Clients usually derive the required bin array indexes from:

- The position range for liquidity operations.
- The active bin and swap direction for swaps.
- Non-contiguous bin IDs for limit order and advanced flows.

## PositionV2

`PositionV2` is the current position account format. It supports:

- `u128` liquidity shares.
- Dynamic extension data for wide positions.
- Position operator authority.
- A dedicated `fee_owner`.
- Lock release point.
- Permissionless operation bits for selected position actions.

The base account stores the default bin count. When a position grows beyond that count, additional `PositionBinData` is appended to the account.

## LimitOrder

Limit orders use a fixed account header plus dynamic `LimitOrderBinData` entries. Each entry stores:

- `amount`
- `age`
- `bin_id`
- `is_ask`

The program can close an empty limit order account with `close_limit_order_if_empty`.

## Token-2022 Accounts

The SDK detects token program ownership and transfer-hook extra account metas for pool tokens and reward tokens. When building custom instructions, include the remaining accounts required by Token-2022 extensions, especially transfer hooks and transfer fees.
