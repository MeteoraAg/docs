## Quote Helpers

DBC supports exact-in, exact-out, and partial-fill quote paths. These map to the `swap2` modes used by modern integrations.

### `quote_exact_in`

Calculates the result of a swap for an exact input amount.

```rust
pub fn quote_exact_in(
    pool: &VirtualPool,
    config: &PoolConfig,
    swap_base_for_quote: bool,
    current_timestamp: u64,
    current_slot: u64,
    in_amount: u64,
    has_referral: bool,
) -> Result<SwapResult2>
```

| Parameter | Meaning |
| --- | --- |
| `pool` | Current `VirtualPool` state. |
| `config` | `PoolConfig` used by the pool. |
| `swap_base_for_quote` | `true` to sell base for quote, `false` to buy base with quote. |
| `current_timestamp` | Current cluster timestamp. |
| `current_slot` | Current cluster slot. |
| `in_amount` | Input amount before DBC fee calculation. |
| `has_referral` | Whether referral fee should be included in the fee split. |

### `quote_exact_out`

Calculates the required input amount for an exact output amount.

```rust
pub fn quote_exact_out(
    pool: &VirtualPool,
    config: &PoolConfig,
    swap_base_for_quote: bool,
    current_timestamp: u64,
    current_slot: u64,
    out_amount: u64,
) -> Result<SwapResult2>
```

Use this for `swap2` exact-out mode, then pass the returned input bound into the transaction with explicit slippage protection.

### `quote_partial_fill`

Calculates a partial-fill quote for an exact input amount.

```rust
pub fn quote_partial_fill(
    pool: &VirtualPool,
    config: &PoolConfig,
    swap_base_for_quote: bool,
    current_timestamp: u64,
    current_slot: u64,
    in_amount: u64,
    has_referral: bool,
) -> Result<SwapResult2>
```

Partial fill is useful near the end of a bonding curve because the final buy may not be able to consume the full input amount without exceeding the migration boundary.

## Return Data

`SwapResult2` includes the values indexers and transaction builders need:

| Field | Use |
| --- | --- |
| `included_fee_input_amount` | Input amount before excluded-fee adjustment. |
| `excluded_fee_input_amount` | Amount that moves the curve after fee handling. |
| `amount_left` | Unused amount in partial-fill flows. |
| `output_amount` | Output token amount. |
| `next_sqrt_price` | Post-swap sqrt price. |
| `trading_fee`, `protocol_fee`, `referral_fee` | Fee split for the swap. |

## CPI Guidance

When calling DBC from another program, generate bindings from the IDL or compare against SDK-built instructions. Account order is especially important for swap and migration flows that include optional accounts, token programs, event authority accounts, or instruction sysvar validation.

Keep user-facing limits in your own instruction arguments:

| Flow | Required bound |
| --- | --- |
| Exact in | Minimum output amount. |
| Exact out | Maximum input amount. |
| Partial fill | Minimum output and expected partial-fill behavior. |

<Warning>
  Never rely on token transfer logs to reconstruct DBC swap price. Use `EvtSwap2`, account state, or the quote helpers so fee-adjusted amounts and `next_sqrt_price` are handled correctly.
</Warning>
