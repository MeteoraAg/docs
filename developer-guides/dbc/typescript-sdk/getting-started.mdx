## Initialization

Initialize the client with a Solana connection and commitment:

```typescript
import { Connection } from "@solana/web3.js";
import { DynamicBondingCurveClient } from "@meteora-ag/dynamic-bonding-curve-sdk";

const connection = new Connection(process.env.RPC_URL!, "confirmed");
const client = new DynamicBondingCurveClient(connection, "confirmed");
```

You can also use the static helper:

```typescript
const client = DynamicBondingCurveClient.create(connection, "confirmed");
```

## Read Pool State

Use `client.state` for indexed-like reads directly from program accounts:

```typescript
import { PublicKey } from "@solana/web3.js";

const poolAddress = new PublicKey("YOUR_POOL_ADDRESS");

const pool = await client.state.getPool(poolAddress);
const config = await client.state.getPoolConfig(pool.config);

const quoteProgress = await client.state.getPoolQuoteTokenCurveProgress(poolAddress);
const baseProgress = await client.state.getPoolBaseTokenCurveProgress(poolAddress);
```

## Example Swap Quote

Quote before building a swap transaction. The SDK mirrors the on-chain curve and fee math.

```typescript
import BN from "bn.js";

const quote = await client.pool.swapQuote({
  virtualPool: pool,
  config,
  swapBaseForQuote: false,
  amountIn: new BN(1_000_000_000), // 1 SOL when quote mint is WSOL
  slippageBps: 100,
  hasReferral: false,
  currentPoint: new BN(0),
});

console.log({
  amountOut: quote.amountOut.toString(),
  minimumAmountOut: quote.minimumAmountOut.toString(),
  nextSqrtPrice: quote.nextSqrtPrice.toString(),
});
```

For new trading integrations, prefer `swap2`/`swapQuote2` when you need partial-fill or exact-out behavior.

## Main Service Areas

| Service | Use |
| --- | --- |
| `client.partner` | Create configs, create partner metadata, claim partner fees, claim pool creation fees, withdraw partner surplus. |
| `client.pool` | Create pools, create config-and-pool flows, create pool with first buy, quote swaps, build swaps. |
| `client.creator` | Create pool metadata, claim creator fees, transfer pool creator, withdraw creator surplus. |
| `client.migration` | Create lockers, migrate to DAMM v1/v2, lock or claim DAMM v1 LP, withdraw migration fees and leftover. |
| `client.state` | Read configs, pools, metadata, fee metrics, progress, tokenomics, and quote math helpers. |

Use the [SDK Functions](/developer-guides/dbc/typescript-sdk/sdk-functions) page for method signatures and the [Example Scripts](/developer-guides/dbc/typescript-sdk/example-scripts) page for full transaction flows.

## Testing The SDK

If you have cloned the SDK repository, start with package-level checks:

```bash
cd packages/dynamic-bonding-curve
bun install
bun run build
bun test
```

Some tests use local validator fixtures and program artifacts. For application integrations, simulate transactions and test on devnet before mainnet.
