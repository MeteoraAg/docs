Dynamic Bonding Curve (DBC) is Meteora's token launch primitive for creating a virtual reserve pool, trading through a configured bonding curve, and migrating finished pools to DAMM v1 or DAMM v2.

For product-level concepts, fee behavior, migration design, and formulas, start with the core product docs.

## Common Flows

| Task | Primary SDK path |
| --- | --- |
| Create a config | `client.partner.createConfig` or `client.pool.createConfigAndPool*` helpers |
| Build a curve | `buildCurve`, `buildCurveWithMarketCap`, `buildCurveWithTwoSegments`, `buildCurveWithLiquidityWeights`, `buildCurveWithMidPrice`, `buildCurveWithCustomSqrtPrices` |
| Create a pool | `client.pool.createPool` |
| Quote a swap | `client.pool.swapQuote` or `client.pool.swapQuote2` |
| Execute a swap | `client.pool.swap` or `client.pool.swap2` |
| Claim trading fees | `client.partner.claimPartnerTradingFee2` or `client.creator.claimCreatorTradingFee2` |
| Migrate to DAMM v1 | `createDammV1MigrationMetadata`, optional `createLocker`, `migrateToDammV1`, then LP lock/claim helpers |
| Migrate to DAMM v2 | Optional `createLocker`, then `migrateToDammV2` |

## Commands

| Task | Command |
| --- | --- |
| Build TypeScript SDK | `cd packages/dynamic-bonding-curve && bun install && bun run build` |
| Run TypeScript SDK tests | `cd packages/dynamic-bonding-curve && bun test` |
| Run Go examples | `go run examples/<file-name>.go` |
| Run program tests | `anchor test` from the `dynamic-bonding-curve` repository |

<Note>
  Some SDK tests depend on local validator fixtures and deployed program artifacts. For production integrations, simulate transactions and test on devnet before sending mainnet transactions.
</Note>
